# Sahulat AI – Project README / Documentation

---

## 1. Overview  

Sahulat AI is a **Flutter‑based mobile application** that lets users browse, book, and pay for services via a card‑booking workflow. The UI follows the **Flip7 design system** (glass‑morphism header, left‑accent gradient bar, dark‑mode support) and is built with **Spec‑Kit**‑generated specifications, **custom Antigravity skills**, and a suite of **MCP (Model‑Context‑Protocol) servers** that expose platform‑specific tooling to the LLM agent.

* **Flutter UI** – Implements the Flip7 design with dark‑mode, glass‑morphism, and gradient accent bars.  
* **Agents & Skills** –  
  * **Spec‑Kit** agents generate and keep the feature spec (`spec.md`) synchronized with implementation.  
  * **Custom Antigravity skills** (e.g., `frontend‑design`, `speckit.*`, `brand‑guidelines`) drive code generation, UI polishing, and documentation.  
* **MCP Servers** – Provide safe, sandboxed access to platform tools:  
  * `android‑adb` – device listing, UI inspection, log streaming.  
  * `firebase‑mcp` – authentication, Firestore, Cloud Functions.  
  * `cloudrun‑mcp` – deployment of backend services, service logs.  
* **Backend** – Stateless Cloud Run services exposing **REST/JSON** APIs for:  
  * **Booking** – create, read, update bookings.  
  * **Payments** – Stripe integration (tokenization, charge).  
  * **User profile** – Firebase Auth + Firestore.

## 3. APIs  

| API | Method | Endpoint | Description |
|-----|--------|----------|-------------|
| **Auth** | POST | `/api/auth/login` | Firebase Auth token exchange |
| **Providers** | GET | `/api/providers` | List available service providers |
| **Booking** | POST | `/api/booking` | Create a new booking (payload: providerId, time, userInfo) |
| **Payment** | POST | `/api/payment` | Process Stripe payment (payload: token, amount) |
| **Summary** | GET | `/api/booking/:id` | Retrieve booking summary for UI screen |

All APIs are documented with OpenAPI JSON generated by the **Spec‑Kit** agent and hosted on the Cloud Run service.

## 4. Agents & Custom Skills Developed  

| Agent / Skill | Purpose | Key Contributions |
|---------------|---------|-------------------|
| **spec‑kit** | Generate/maintain `spec.md` from natural language feature description | Auto‑syncs UI spec with code via `speckit` pipeline |
| **speckit.plan** | Produce implementation plan & `tasks.md` | Drives task ordering, dependency analysis |
| **speckit.implement** | Executes tasks & creates PR‑ready code | Ensures ironclad anti‑regression protocol |
| **frontend‑design** (custom) | Apply Flip7 visual language, glass‑morphism, dark‑mode | Generates high‑fidelity UI components |
| **brand‑guidelines** (custom) | Enforce Anthropic brand colors & typography across assets | Guarantees consistent branding |
| **android‑cli** (custom) | Automates Android SDK checks, emulator launch, log capture | Used for UI validation on device |
| **firebase‑mcp** (custom) | Wraps Firebase Auth/Firestore operations for the agent | Simplifies backend data handling |
| **cloudrun‑mcp** (custom) | Deploys backend containers, fetches logs | Enables one‑click deploy from the agent |

## 5. Integration Details  

1. **UI ↔️ Backend** – Flutter uses the generated `api_client.dart` (OpenAPI codegen) to call Cloud Run endpoints.  
2. **MCP ↔️ Agents** – Each MCP server is registered under `C:\Users\MEHDI\.gemini\antigravity\mcp\<serverName>`. Agents invoke them via `call_mcp_tool` (e.g., `adb_devices`, `firebase_login`).  
3. **Spec‑Kit ↔️ Code** – The `implementation_plan.md` created by `speckit.plan` references concrete file paths; the `speckit.implement` skill executes `replace_file_content`/`multi_replace_file_content` to modify source files.  
4. **Testing** – `webapp-testing` skill runs Playwright against the local emulator UI, ensuring Flip7 design compliance and functional flow.

## 6. How to Run the Project  

```bash
# 1️⃣ Clone the repo
git clone https://github.com/mehdinathani/SahulatAI.git
cd SahulatAI

# 2️⃣ Install Flutter dependencies
flutter pub get

# 3️⃣ Run the Android emulator (MCP android‑adb will launch if needed)
flutter emulators --launch pixel_5_api_33

# 4️⃣ Launch the app
flutter run

# 5️⃣ Deploy backend (requires Google Cloud SDK)
cd backend
gcloud run deploy sahulat-backend --source . --region us-central1 --allow-unauthenticated
```

*All deployment steps are orchestrated by the `cloudrun‑mcp` skill, which the agent can invoke directly from the UI.*
